# 🚀 退休財務模擬計算機 - 多頁面與收支細化重構清單

## 階段一：多頁面架構與狀態管理重構 (Multipage & Session State)
- [x] **Task 1: 導入 Streamlit 多頁面/多視窗導覽**
    * 利用新版 `st.navigation` 將應用程式拆分為：`📊 退休資產模擬看板` 與 `⚙️ 財務參數設定中心`。
- [x] **Task 2: 初始化 Session State 變數池與「立即計算」機制**
    * 建立全域狀態管理。在設定頁面最下方加入 `st.button("📊 儲存配置並執行模擬")` 觸發按鈕。
    * 當使用者調整 Slider 時**不即時重算**以維持網頁流暢度，點擊按鈕後才一次性將數據寫入 `st.session_state`並導向看板。

## 階段二：設定頁面「子標籤頁」與全面中文化開發 (Granular UI & Localization)
* [x] **Task 3: 全面繁體中文化 (Localization)**
    * 將程式中所有的 Sliders, Number Inputs, Metric Labels, 圖表文字等全部替換為繁體中文。
* [ ] **Task 4: 在設定頁面建立「子分頁 (st.tabs)」與預設切換器**
    * 在設定頁面內建立四個小分頁（💰 收入配置、🍔 日常基本支出、🎨 娛樂與雜支、🏠 里程碑重大支出）。
    * **【進階實作】**：在「收入配置」中加上「套用台積電 31-33 職等預設薪資路徑」Checkbox。勾選時，手動調薪 Slider 變更為 Disabled（停用），直接套用 CSV 基底陣列。
    * **【新功能加入】**：在「收入配置」中加上「啟用台灣勞退/勞保年金估算」切換開關，並允許輸入「預期開始領取年齡」（預設 60 或 65 歲）與「預估每月年金（元）」。

## 階段三：核心模擬引擎升級 (Core Logic Upgrade)
* [ ] **Task 5: 重構 `run_simulation` 計算引擎（分拆通膨邏輯）**
    * 升級計算迴圈，將支出拆解為動態加總。
    * **【進階實作】**：日常與娛樂支出逐年計算通膨放大；但**房貸本息攤還金額（如 66.8 萬）一經開貸即固定，不吃歷年通膨率**。
* [ ] **Task 6: 整合政府勞退/勞保年金發放金流**
    * **【新功能加入】**：在核心 `for` 迴圈中加入年齡/年份判斷。當模擬年份達到使用者設定的領取年齡（例如 2060 年起），**自動在當年的「總收入」中疊加該筆政府年金現金流**，用以降低退休後對核心資產的提領壓力。
* [ ] **Task 7: 精準化所得稅計算（升級 `calculate_tax`）**
    * 導入台灣綜合所得稅率級距（5%, 12%, 20%, 30%, 40%）與基本扣除額（約 44.6 萬）計算函式，自動精算歷年「稅後實質薪資」。
